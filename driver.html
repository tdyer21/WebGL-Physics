Planning Document
For our project weâ€™re planning to create an interactive information display for a product (maybe ourselves). We want to move the user from 3d scene to 3d scene by moving the camera to different scenes in a 3d world or rendering new ones, with the user having the ability to create and destroy objects within the scene. Within each scene we plan to exhibit a host of various effects and user interactions. We plan to have different text in each scene with a clickable hud for navigation. The HUD could be a shape or just a < > html text to navigate from one scene to the next. To navigate we plan to use animation with keyframing. We are going for a clean and simple aesthetic with warm colors, low poly object designs, and soft lighting. We want the experience to feel friendly and approachable. Project Quantum is designed to fluidly move users between 3d scenes to help improve the users understanding of a product or idea.
Description of each of (team members x2) advanced methods you have implemented. Interactive HUD: We are going to use this for navigation. There can be a markers or buttons to click to get from one screen to the next. We will be able to use this to make sure the user can get around the project. Text onto objects: We wanted to show how to do the interactions we created so we put text on the walls to show how to do everything. Collision Detection: We added collision detection using our own math, against the wall and the ceilings and floors. Object Physics (gravity + friction): We simulated acceleration due to gravity to create semi-realistic physics for bouncing spheres, and friction so that the spheres eventually stop rolling. (done without the use of any outside library, built from scratch). Under water Physics: simulated spheres with bouyancy in the second scene. Color Picking: We made it so you can choose the color of the objects. KeyFraming: Moving the camera using keys insead and zooming around.
